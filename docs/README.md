# Документация проекта ИРУ

Вся пояснительная документация вынесена из корня проекта в папку `docs/`, чтобы рабочие файлы приложения были отделены от текстовых инструкций.

## Документы

### 1. Описание проекта

```text
docs/PROJECT.md
```

Содержит:

- описание сайта и CRM;
- структуру проекта;
- список API;
- таблицы базы данных;
- основные команды.

### 2. Первый запуск

```text
docs/FIRST_RUN.md
```

Самая простая инструкция для первого запуска через Railway.

### 3. Production-развёртывание

```text
docs/DEPLOYMENT.md
```

Содержит:

- подключение домена;
- DNS;
- переменные окружения;
- Docker Compose;
- PM2;
- Nginx;
- SSL;
- backup базы данных;
- финальный чеклист.

### 4. Запуск на Vercel

```text
docs/VERCEL.md
```

Содержит инструкцию по исправлению ошибки `DATABASE_URL is required` и настройке переменных окружения Vercel.

### 5. Админ-панель CRM

```text
docs/CRM_ADMIN.md
```

Содержит адрес входа, email администратора и список переменных Vercel для защиты CRM.

### 6. Личный кабинет и админ-панель

```text
docs/ACCOUNT_AND_ADMIN.md
```

Содержит новую схему работы: клиент отправляет заявку в личном кабинете, администратор видит её в админ-панели.

## Рабочие файлы проекта

В корне проекта теперь остаются только технические файлы приложения:

- `package.json`;
- `next.config.ts`;
- `tsconfig.json`;
- `drizzle.config.json`;
- `.env.example`;
- `Dockerfile`;
- `docker-compose.prod.yml`;
- `ecosystem.config.cjs`;
- `src/`;
- `public/`;
- `deploy/`.
